# SMOTE Keep Synthetic Feature Documentation

## Overview

The `smote_keep_synthetic` parameter allows you to control whether synthetic samples generated by SMOTE (Synthetic Minority Over-sampling Technique) are included in the model output's `source.data`.

## Default Behavior

**By default (`smote_keep_synthetic = TRUE`):**
- `source.data` contains SMOTE-enhanced training data with synthetic samples
- A `synthesized` column is added to mark real vs. synthetic samples
- Output row count will be larger than input due to synthetic samples
- Test data remains pure (no synthetic samples)
- This allows you to see exactly what data was used for training

## When to Use `smote_keep_synthetic = FALSE`

Use `smote_keep_synthetic = FALSE` when you want to:

1. **Hide synthetic samples** - Keep output clean by showing only original data
2. **Match input/output row counts** - Easier to trace back to original rows
3. **Simplify downstream analysis** - No need to filter out synthetic samples
4. **Reduce output size** - Smaller datasets without synthetic samples

## How It Works

### With `smote_keep_synthetic = TRUE` (Default):

```r
model_df <- data %>%
  build_lm.fast(
    target, predictor1, predictor2,
    test_rate = 0.3,
    smote = TRUE,
    smote_keep_synthetic = TRUE  # Include synthetic samples (default)
  )

source_data <- model_df$source.data[[1]]
```

**Result:**
- `source.data` contains: **SMOTE-enhanced training data + original test data**
- A `synthesized` column is added to mark samples:
  - `TRUE` = Synthetic sample created by SMOTE
  - `FALSE` = Original real sample from your data
- Row count will be **larger** than input due to synthetic samples
- Test data remains pure (no synthetic samples)

### With `smote_keep_synthetic = FALSE`:

```r
model_df <- data %>%
  build_lm.fast(
    target, predictor1, predictor2,
    test_rate = 0.3,
    smote = TRUE,
    smote_keep_synthetic = FALSE  # Hide synthetic samples
  )

source_data <- model_df$source.data[[1]]
```

**Result:**
- `source.data` contains: **original data only**
- No `synthesized` column
- Row count matches input row count
- Synthetic samples were used for training but not stored in output

## Supported Model Types

The `smote_keep_synthetic` parameter is available in:

| Function | Model Type | Support |
|----------|------------|---------|
| `build_lm.fast()` | Linear/Logistic Regression | ✓ Yes |
| `exp_xgboost()` | XGBoost | ✓ Yes |
| `exp_lightgbm()` | LightGBM | ✓ Yes |
| `calc_feature_imp()` | Ranger (Random Forest) | ✓ Yes |
| `exp_rpart()` | Decision Trees | ✓ Yes |

## Examples

### Example 1: Inspecting Synthetic Samples (Default Behavior)

```r
library(dplyr)

# Create imbalanced data
data <- data.frame(
  x1 = rnorm(100),
  x2 = rnorm(100),
  target = c(rep(FALSE, 90), rep(TRUE, 10))  # 10% minority
)

# Build model with SMOTE (synthetic samples included by default)
model_df <- data %>%
  build_lm.fast(
    target, x1, x2,
    model_type = "glm",
    smote = TRUE
  )

# Inspect the data
source_data <- model_df$source.data[[1]]

cat("Total rows:", nrow(source_data), "\n")
cat("Real samples:", sum(!source_data$synthesized), "\n")
cat("Synthetic samples:", sum(source_data$synthesized), "\n")

# View some synthetic samples
synthetic_samples <- source_data %>% 
  filter(synthesized == TRUE) %>%
  head(10)
print(synthetic_samples)
```

### Example 2: With Test Mode (Default includes synthetic samples)

```r
# With test mode enabled (synthetic samples included by default)
model_df <- data %>%
  build_lm.fast(
    target, x1, x2,
    model_type = "glm",
    test_rate = 0.3,
    smote = TRUE
  )

source_data <- model_df$source.data[[1]]
test_index <- model_df$.test_index[[1]]

# Verify test data is pure (no synthetic samples)
test_data <- source_data[test_index, ]
cat("Synthetic samples in test data:", sum(test_data$synthesized), "\n")  # Should be 0

# Check training data composition
train_data <- source_data[-test_index, ]
cat("Real training samples:", sum(!train_data$synthesized), "\n")
cat("Synthetic training samples:", sum(train_data$synthesized), "\n")
```

### Example 3: Comparing with and without synthetic samples

```r
# Model 1: Keep synthetic samples (default)
model1 <- data %>%
  exp_xgboost(
    target, x1, x2,
    smote = TRUE,
    nrounds = 10
  )

# Model 2: Hide synthetic samples
model2 <- data %>%
  exp_xgboost(
    target, x1, x2,
    smote = TRUE,
    smote_keep_synthetic = FALSE,
    nrounds = 10
  )

cat("Model 1 output rows:", nrow(model1$source.data[[1]]), "\n")
cat("Model 2 output rows:", nrow(model2$source.data[[1]]), "\n")

# Both models use the same SMOTE-enhanced data for training
# Only the output differs
```

## Important Notes

### Test Data Purity
- **Test data NEVER contains synthetic samples**, regardless of `smote_keep_synthetic` setting
- SMOTE is only applied to training data after the train/test split
- This ensures valid model evaluation on real, unseen data

### Row Indexing
When `smote_keep_synthetic = TRUE`:
- `.test_index` points to test data positions in the SMOTE-enhanced `source.data`
- Test data is always at the end of `source.data`
- Use care when manually indexing rows

### Predictions
- Predictions work correctly with both settings
- When `smote_keep_synthetic = TRUE`, predictions are generated for all rows (including synthetic samples)
- When `smote_keep_synthetic = FALSE`, predictions match the original data

### Column `synthesized`
The `synthesized` column is only added when `smote_keep_synthetic = TRUE` **AND** SMOTE is actually applied (i.e., for binary classification with imbalanced classes).

## Performance Considerations

Using `smote_keep_synthetic = TRUE`:
- Slightly increases memory usage (storing synthetic samples)
- May slow down prediction functions slightly (more rows to predict)
- No impact on model training speed or quality

## Best Practices

1. **Use default `TRUE` for transparency** - See exactly what data was used for training
2. **Use `FALSE` for cleaner output** - When you don't need to see synthetic samples
3. **Filter synthetic samples if needed** - Use `filter(!synthesized)` in downstream analysis
4. **Document your choice** - Make it clear in your code whether synthetic samples are included

## Troubleshooting

### Issue: `synthesized` column missing

**Possible causes:**
1. SMOTE was not actually applied (check if target is binary classification)
2. Classes are already balanced (SMOTE skipped)
3. Not enough minority samples for SMOTE (needs at least `k+1` samples, default k=5)
4. `smote_keep_synthetic = FALSE` was explicitly set

**Solution:** Check your target variable and class distribution.

### Issue: More rows than expected in output

**This is normal with the default behavior (`smote_keep_synthetic = TRUE`)**. SMOTE generates synthetic samples to balance classes, so output will have more rows than input. To get original row count only, set `smote_keep_synthetic = FALSE`.

### Issue: Predictions include synthetic samples

**This is expected with the default behavior**. The predictions will include rows for synthetic samples. Filter them out if needed:

```r
predictions <- model_df %>% 
  prediction_binary(data = 'training_and_test')

# Keep only real samples
real_predictions <- predictions %>% 
  left_join(source_data %>% select(row_id, synthesized)) %>%
  filter(!synthesized)
```
